# PaperOrchestra: A Multi-Agent Framework for Automated AI Research Paper Writing

- alphaXiv: https://www.alphaxiv.org/abs/2604.05018
- Source paper: https://arxiv.org/abs/2604.05018

The translation of research ideas and experimental findings into a formal, submission-ready manuscript is a resource-intensive stage of the scientific process. As Large Language Models (LLMs) take a more active role in scientific discovery, there is an increasing demand for systems that can automate this translation. While existing autonomous research frameworks often include writing components, these modules are typically tied to internal experimental loops, making them less effective for processing external, unstructured materials provided by human researchers. Furthermore, many current automated systems struggle with synthesizing comprehensive literature reviews or generating conceptual diagrams that explain complex methodologies.

PaperOrchestra is a multi-agent framework developed by researchers at Google to address these limitations. It functions as a standalone system capable of transforming unstructured pre-writing materials—such as high-level idea summaries and raw experimental logs—into complete LaTeX manuscripts. By decoupling the writing process from the experimental phase, the framework offers a flexible tool for researchers to convert early-stage outputs into structured academic papers.

## Task Formulation and Input Requirements

The end-to-end writing task is formulated as a mapping function that transforms a set of unconstrained inputs into a finalized submission package. The system processes five primary input components:

1.  **Idea Summary ($I$):** A brief overview of the methodology, theoretical foundations, and proposed contributions. This can be sparse (high-level) or dense (containing formal definitions).
2.  **Experimental Log ($E$):** A compilation of raw results, ablation studies, and performance metrics.
3.  **LaTeX Template ($T$):** The specific formatting files required by the target venue.
4.  **Conference Guidelines ($G$):** Mandatory requirements for submission.
5.  **Figures ($F$):** Optional pre-existing visual assets. If no figures are provided ($F = \emptyset$), the system autonomously synthesizes them.

The output is a finalized submission package $P$, which includes the source LaTeX file $P_{\text{tex}}$ and the rendered PDF $P_{\text{pdf}}$. The relationship is expressed as:

$$
P = f(I, E, T, G, F)
$$

## The Multi-Agent Architecture

PaperOrchestra utilizes a specialized multi-agent system where different agents handle specific aspects of the manuscript generation process. This modular approach allows for targeted optimization of complex tasks like literature search and visual synthesis.

### 1. Outline Agent
The process begins with the Outline Agent, which synthesizes the raw materials into a comprehensive JSON outline. This outline serves as the architectural blueprint for the paper. It includes a visualization plan (specifying plot types and data sources), a literature search strategy (defining macro-level context for the introduction and micro-level clusters for the related work), and a section-level plan with content bullets and citation hints.

### 2. Plotting Agent (PaperBanana)
Visual content is generated by the Plotting Agent using a module called PaperBanana. Unlike systems that only produce data plots from code, PaperOrchestra generates both statistical plots and conceptual diagrams (e.g., architectural diagrams). It employs a closed-loop refinement system where a Vision-Language Model (VLM) criticizes generated visuals and iteratively revises the code to resolve artifacts and ensure the images align with the text.

### 3. Literature Review Agent
A significant challenge in automated writing is the production of "Related Work" sections that are analytical rather than merely descriptive. The Literature Review Agent uses a hybrid discovery pipeline. It identifies candidate papers via web search and authenticates them using the Semantic Scholar API to retrieve metadata and enforce temporal cutoffs. It then compiles a citation registry, generates a BibTeX file, and drafts the Introduction and Related Work sections by positioning the current research against the existing landscape.

### 4. Section Writing Agent
This agent drafts the core technical sections, including the methodology, experiments, and conclusion. It extracts numerical data from the experimental log to construct LaTeX tables and integrates the figures generated by the Plotting Agent. The agent ensures that the narrative remains consistent with the section-level outline and the established citation bank.

### 5. Content Refinement Agent (AgentReview)
The final stage involves an iterative optimization process that mimics human peer review. The Content Refinement Agent uses a simulated review framework to evaluate the initial draft. Revisions are made based on feedback regarding clarity, presentation, and scientific soundness. A version control mechanism is implemented where revisions are only accepted if they improve the overall score; otherwise, the system reverts to a previous iteration or halts upon reaching a limit.

## PaperWritingBench: A New Standard for Evaluation

To rigorously assess the performance of PaperOrchestra, the researchers introduced PaperWritingBench. This benchmark dataset consists of 200 accepted papers from top-tier AI conferences (CVPR 2025 and ICLR 2025). 

The benchmark was created by reverse-engineering these papers into their constituent raw materials. An LLM was used to distill the methodology into idea summaries and extract findings into experimental logs while stripping away citations and explicit figure references. This process ensures that evaluation is focused strictly on the writing agent's ability to reconstruct a professional paper from raw data, without relying on the original author's narrative structure.

## Empirical Findings and Performance

PaperOrchestra was evaluated against several baselines, including a monolithic single-agent LLM and the state-of-the-art AI Scientist-v2 pipeline. The evaluation utilized both automated "autoraters" and blind human reviews by AI researchers.

### Literature Synthesis and Citation Rigor
One of the primary findings was the framework's ability to mitigate under-citation. Traditional autonomous systems often produce superficial reviews with very few citations. In contrast, PaperOrchestra generated an average of 45–48 citations per paper, approaching the human ground truth average of 59. 

In terms of citation coverage, the system showed significant improvements in recall for $P_0$ (Must-Cite) and $P_1$ (Good-to-Cite) references. The "Literature Review Quality" metric, which evaluates coverage, relevance, and critical analysis, showed that PaperOrchestra achieved absolute gains of over 30% compared to existing autonomous baselines.

### Overall Manuscript Quality
Using AI-based peer review frameworks, PaperOrchestra achieved simulated acceptance rates of 84% for CVPR and 81% for ICLR. These rates are comparable to the human-written ground truth papers in the benchmark. Side-by-side (SxS) comparisons conducted by LLM judges showed that PaperOrchestra won 88%–99% of matchups against baselines in literature review quality and 39%–86% in overall paper quality.

### Human Validation
Human evaluations correlated significantly with the automated metrics ($r = 0.6458$ and $\rho = 0.6355$ for overall quality). Human reviewers rated PaperOrchestra as superior to AI baselines in over 50% of literature review matchups. Notably, the system achieved a 43% tie or win rate when compared directly against the original human-authored manuscripts in literature synthesis, suggesting a high degree of technical coherence.

## The Role of Iterative Refinement

A critical component of the framework's success is the Content Refinement Agent. Ablation studies demonstrated that the refinement loop is essential for reaching submission-ready quality. Manuscripts that underwent iterative refinement dominated unrefined drafts in 80% of matchups. This step led to a 19–22% increase in simulated acceptance rates, underscoring the importance of self-critique and revision in the scientific writing process.

Furthermore, the autonomous visual generation module (PaperBanana) was found to be highly competitive. Despite the inherent difficulty of generating technical diagrams without human guidance, the Plotting Agent's outputs achieved ties or wins in 51%–66% of matchups against human-authored figures when evaluated for their ability to support the paper's narrative.

## Implications for the Research Community

PaperOrchestra represents a shift toward more flexible and high-fidelity automated scientific writing. By focusing on deep literature synthesis and visual diversity, it addresses the most common criticisms of current AI-generated content—namely, that it is often generic or visually sparse.

The framework is intended to serve as an assistive tool rather than a replacement for human researchers. It automates the structured drafting, formatting, and literature sourcing phases, potentially allowing researchers to focus on high-level conceptual work. However, the authors maintain that human accountability remains paramount. Users are responsible for verifying the accuracy of the experimental logs and the validity of the generated methodology to prevent the propagation of hallucinations or technical errors.

The introduction of PaperWritingBench provides the community with a standardized tool for ongoing research in this domain. Future developments may involve the integration of more diverse research artifacts, such as raw data files or experimental codebases, to further enhance the system's ability to generate precise and data-driven scientific narratives.
